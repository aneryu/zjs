# qjs-faithful frame structural alignment — design

## Aligned representation

TARGET REPRESENTATION (qjs-faithful, Zig-substituted where alloca is unavailable):

(A) SLIM FRAME HEADER — a 9-field hot core that maps 1:1 onto qjs JSStackFrame (quickjs.c:407-420), replacing the 27-field frame.zig:225-251 struct. Fields, all HOT (read/written per-opcode or once at setup, read by cold ops):
  function: *const Bytecode        (= sf->cur_func->b source for code.ptr/len)
  pc: usize                        (= sf->cur_pc; mirrored to reg_ip)
  args: []JSValue                  (= sf->arg_buf; BORROWED from caller region when argc>=arg_count)
  locals: []JSValue                (= sf->var_buf)
  var_refs: []JSValue              (= sf->var_refs; ALIASED to functionCapturesSlot().*, not copied)
  cur_func: JSValue                (= sf->cur_func; NON-OWNING cast-store, no take/free)
  arg_count: usize                 (= sf->arg_count)
  cold: ?*FrameCold                (NEW: null on the common plain call; heap side-struct only when a cold feature is live)
  open_var_refs: []?*VarRef        (frame's OWN open closures over locals/args; faithful to sf->var_refs NULL-fill+close, KEEP)
  prev/backtrace: reuse the Entry-chain link (no separate field — see D)

  REMOVED FROM HOT CORE (qjs keeps NONE of these in sf): this_value, this_value_owned, constructor_this_value, constructor_this_value_owned, new_target, arguments_object, original_args (slice handle stays only inside FrameCold), storage_values, storage_on_heap, the 4 eval_* fields + eval_var_refs_republished, the 4 global_lexical_sync_* fields. ~18 of 27 fields leave the hot core.

(B) FrameCold side-struct (heap-allocated lazily, reached only via frame.cold!=null), holding the three zjs-only feature blocks + the arena bookkeeping handle:
  block A (arguments): arguments_object, original_args, actual_arg_count
  block B (global-lexical-sync): sync_env, sync_slots, sync_indices, sync_checked
  block C (direct-eval): eval_local_names/slots, eval_var_ref_names, eval_var_refs, republished
  storage: storage_values, storage_on_heap (touched only at setup/teardown, never per-opcode)
  constructor_this_value (+owned) for derived-class ctors.
  Generators/async MUST keep frame.cold reachable across suspension (vm_gen_async writes sync_* at suspend).

(C) RAW-SP OPERAND STACK over the slab window — BIFURCATED, not unified.
  HOT inline regime: the operand stack IS the slab sub-slice (frame.zig:76 stack = slab_values[cursor..], identical to qjs stack_buf = var_buf+var_count, quickjs.c:17860). The dispatch loop ALREADY runs raw reg_base/reg_sp (zjs_vm.zig:850-851,901-903) with *sp++/*--sp, no bounds checks (zjs_vm.zig:927-928,1627-1628). The 5-field Stack object is reduced to a {ptr,len} window on the Entry; syncDown collapses to just the pc save (frame.pc = reg_ip-code.ptr), the operand-length re-derivation disappears, because reg_sp stays authoritative across the threaded post-call resume.
  COLD growable regime (top-level/native re-entry/generator/async): the full heap-backed Stack object with {capacity,limit,memory,arena_window} is RETAINED unchanged (stack.zig). Generators transfer {values,capacity,buffer-ownership} into the generator object (vm_gen_async.zig:82-83) and physically cannot use a borrowed slab window. Do NOT unify these.

(D) SINGLE CARVE — KEEP AS-IS. frame.zig:67 does ONE arena.carve partitioned by cursor arithmetic (args/original_args/locals/stack/var_refs/open_var_refs), byte-for-byte faithful to qjs's single alloca + pointer partition (quickjs.c:17846,17856-17864). mark/restore (inline_calls.zig:297,674) is the irreducible Zig substitute for alloca's free C-unwind (Zig has no variadic alloca). allocHeap (frame.zig:100) is a load-bearing safer-than-qjs fallback. The ONLY change: fill the Frame/Entry window fields via OUT-POINTER instead of returning FrameSlab by value (88B) then re-bundling into FrameStorageWindows (inline_calls.zig:389-395) — collapse the two descriptor structs into one out-filled view.

(E) DISPATCH ACCESS PATTERN — threaded post-call resume that avoids the per-call cold re-entry. Today both pushCall (inline_calls.zig:258) and popTeardown (inline_calls.zig:661) set machine.switched=true, forcing EVERY call AND EVERY return through the full cold prologue: ~17-field switched frame-identity reload (zjs_vm.zig:855-895) + 5-register reload (:899-903) + pc bound check (:904) + interrupt poll (:911) + fetch. Even Path B's recursive call does stack.pushOwnedAssumeCapacity(ret); continue; (zjs_vm.zig:1855) — a bare cold re-entry that reloads registers the caller never lost. Target: make recursion (Path B) the structural model so the caller's reg_ip/reg_base/reg_var_buf survive the call, then after callInlineRecursive returns do qjs's quickjs.c:18199-18201+BREAK: reg_sp -= consumed; reg_sp[0]=ret; reg_sp+=1; opc=reg_ip[0]; reg_ip+=1; continue :sw opc; — NO machine.switched, NO prologue, NO poll. Move the interrupt poll to function ENTRY (setupInlineEntry / runFrameLoop top), matching qjs's single js_poll_interrupts at entry (quickjs.c:17787); backward-branch arms already poll (zjs_vm.zig:1595,1638) so loops can't starve. Exception/non-plain-callee (eval/generator/C-fn/single-step) stay on the cold path.

(F) BACKTRACE FROM THE FRAME CHAIN. Delete the parallel ActiveBacktraceFrame push/pop per call (inline_calls.zig:302-303,675; context.zig:797-806) and derive backtraces by walking the Machine Entry-chunk chain (depth-1..0 then l0_frame) exactly as qjs walks rt->current_stack_frame->prev_frame (quickjs.c:7571). The Entry chain ALREADY is a walkable frame chain — the resolver reads everything from *Frame (vm_exception_ops.zig:337-356). Keep lazy materialization (on-throw only via snapshotBacktraceFrames) and the backtrace_barrier stop (vm_exception_ops.zig:354). The L0 recursive boundary (zjs_vm.zig:518) must feed the same walk.

INVARIANTS PRESERVED THROUGHOUT (non-negotiable, from facet 6): refcount-only frame liveness (traceRoots never walks the frame/Entry chain — runtime.zig:1166); open VarRefs alias stable slab slots and closeOpenVarRefs runs BEFORE slot free (frame.zig:573 before releaseValueSlice); the borrowed-vs-owned free ledger must flip in lockstep with each borrow optimization (skip-free var_refs when aliased, skip-free cur_func when borrowed) or test262+force-GC breaks with double-free/leak; catch-marker discipline + abrupt-completion iterator close survive on the raw-sp window.

## Whole-chain changes

- **Frame struct split (keystone)**: Replace the 27-field Frame (frame.zig:225-251) with a ~9-field hot core + a heap FrameCold side-struct (blocks A/B/C + storage handle + ctor-this). Frame.init zeroes 9 fields not 27; deinitInlineCall frees the hot core then frees frame.cold only when non-null. (src/exec/frame.zig:225-251,253-255,500-549; src/exec/inline_calls.zig:116,300-301) — ripple: Every reader of this_value/new_target/constructor_this/arguments_object/eval_*/global_lexical_sync_*/storage_* now goes through frame.cold (13x object_ops, 11x call_runtime cur_func reads, eval_ops, vm_call, class_init, vm_gen_async, slot_ops sync path). Each must lazily allocate frame.cold on write and treat null as the default. Because Frame is inline in Entry, the inline-call slab shrinks ~5x.
- **Lazy this / borrowed cur_func / borrowed new_target**: Drop the eager coerceCallThis + persisted this_value/_owned (inline_calls.zig:308-357); pass this/new_target as call params, coerce this lazily at the existing op.push_this handler (zjs_vm.zig:2163). Make cur_func a NON-OWNING cast-store (drop takeSourceSlot at inline_calls.zig:345 and the unconditional free at frame.zig:503), rooted by the caller's still-live operand region for the call duration. (src/exec/inline_calls.zig:308-357,345,469-473; src/exec/frame.zig:503; src/exec/zjs_vm.zig:2163; src/exec/call_runtime.zig:295-310 (coerceCallThis moves to push_this)) — ripple: op.push_this must now do the sloppy-undefined->global / primitive->ToObject coercion qjs does at quickjs.c:17933. The caller MUST keep the callable slot alive until the callee returns (qjs invariant) — couples to the call-region cleanup: cleanupSource must not free the callable slot early. take_receiver_as_this + boxed_this defer-free branches disappear. Ownership ledger in deinitInlineCall must flip cur_func from always-free to never-free in lockstep (facet 6 constraint).
- **var_refs alias instead of copy+retain**: Alias frame.var_refs = functionCapturesSlot().* (already read at inline_calls.zig:287) and skip the per-element value.dup() in initFrameVarRefs and the per-element free in teardown. Keep the frame's OWN open_var_refs fill/close (faithful). (src/exec/vm_call.zig:152-245 (initFrameVarRefs); src/exec/frame.zig:506,511 (skip releasing var_refs); src/exec/inline_calls.zig:450-452) — ripple: Removes O(captures) refcount traffic per call (hot for fib's self-ref). The var_refs slab sub-slice carve becomes unnecessary for the borrowed case (carve var_ref_count=0 when aliasing). Teardown must NOT releaseValueSlice over var_refs when borrowed; the eval-merge path (mergeEvalBindings) still owns its merged slice and keeps its own free.
- **Raw-sp operand stack (bifurcation)**: Reduce the inline Entry's Stack to a {ptr,len} window; keep reg_sp authoritative across the threaded post-call resume so syncDown collapses to just the pc save. Retain the full 5-field Stack only for the cold growable/generator/native-entry regime. (src/exec/stack.zig:6-14; src/exec/zjs_vm.zig:84-94 (syncDown), :899-903 (reload), ~150 syncDown sites; src/exec/inline_calls.zig:396) — ripple: ~165 cold-handler call sites (vm_value 165 refs, vm_call 67, object_ops 48, vm_property_field 48, etc.) currently take a Stack*; the cheap intermediate is to keep them mutating stack.values in place while the loop keeps reg_sp authoritative (already ~90% the case) and only collapse syncDown. Full raw-sp signature rewrite is the expensive long-tail (do last). Generator suspend/native-entry growth UNCHANGED.
- **Threaded post-call/post-return resume + entry-only poll**: Make Path B (build.zig:11 zjs_recursive_call) the default. After callInlineRecursive returns, replace stack.pushOwnedAssumeCapacity(ret); continue; (zjs_vm.zig:1855,1934) with reg_sp[0]=ret; reg_sp+=1; opc=reg_ip[0]; reg_ip+=1; continue :sw opc; (mirror quickjs.c:18199-18201+BREAK), no machine.switched. Move interrupt poll from the prologue (zjs_vm.zig:911) to function entry. (build.zig:11; src/exec/zjs_vm.zig:1845-1856,1924-1935,911; src/exec/inline_calls.zig:258,661 (stop forcing switched for the plain shape)) — ripple: Eliminates ~17-field switched reload + 5-reg reload + bound check + poll per call AND per return (zjs_vm.zig:855-911). Exception case keeps handleCatchableRuntimeError (must syncDown first). The op.return arms (zjs_vm.zig:1419-1436) under recursion return up the Zig stack instead of machine.popReturn. Path A (loop-in-place Machine) becomes a fallback only; its switched reload stays correct for it. Interrupt-responsiveness must be verified (entry + backward-branch poll covers loops).
- **Single backtrace chain**: Delete pushActiveBacktraceFrame/popActiveBacktraceFrame per call (inline_calls.zig:302-303,675); derive backtraces by walking the Entry-chunk chain + l0_frame, like qjs walks prev_frame. (src/exec/inline_calls.zig:215-219 (keep resolver/data seed),302-303,675; src/core/context.zig:40-44,797-834; src/exec/vm_exception_ops.zig:337-356) — ripple: snapshotBacktraceFrames must iterate the Machine depth/chunks instead of current_backtrace_frame. backtrace_barrier stop + lazy materialization preserved. The recursive Path B boundary (zjs_vm.zig:518) needs a way to chain stack-local entries into the walk (a thread-local current-entry pointer, qjs's rt->current_stack_frame analog) — this is the one new piece recursion requires.
- **Carve descriptor collapse + teardown gating**: Fill Frame/Entry windows via out-pointer instead of by-value FrameSlab + re-bundled FrameStorageWindows (inline_calls.zig:389-395). Gate the 7-step teardownInlineEntry (inline_calls.zig:668-677) behind a 'simple frame' flag so the no-eval/no-profile common call skips eval_snapshot.deinit/freeEvalResources/backtrace-pop. (src/exec/frame.zig:51-98,100-148; src/exec/inline_calls.zig:367-396,668-677) — ripple: Keep single-bump carve + mark/restore + original_args gate + allocHeap fallback intact (all faithful/load-bearing). Teardown collapses toward qjs's one value-free loop + one prev-pointer store + slab rewind as each setup op aligns. No-double-free invariant (watermark restore reclaims slab; storage_values freed only if storage_on_heap) MUST be preserved.

## Expected ceiling

Realistic floor: ~330-400 insn/call (1.2-1.5x qjs's 271), NOT 271. Reasoning by residual: (1) The single-carve, mark/restore, undefined-fill, open-var-ref close, and original_args gate are ALREADY faithful (~8 insn carve) — no further gain there. (2) Stages 1-4 remove the genuinely structural tax: per-call cold re-entry (~100 insn/node), this/cur_func/var_refs ownership round-trips, the 18 dead Frame fields, the parallel backtrace, and the descriptor double-copy. That recovers the bulk of the 5.4x. (3) IRREDUCIBLE RESIDUAL that keeps zjs above 271: (a) Zig has no variadic alloca, so the VmStackArena carve + Mark{chunk,used} + explicit restore is a few insn/call that qjs gets free from the C-stack unwind (quickjs.c:20710) — structurally permanent. (b) Zig codegen vs the hand-tuned C + computed-goto: even the threaded `continue :sw` is a labeled-switch the optimizer lowers, not a literal `goto *dispatch_table[*pc++]`; bounds/null handling and Zig's slice {ptr,len} fatness add a constant factor. (c) Refcount DISCIPLINE: zjs's per-slot .dup/.free balance is the same shape as qjs but Zig's requiresRefCount branch + the slim-frame's lazy-cold-alloc null-checks add a few insn the C path folds away. (d) The growable-Stack bifurcation means the inline path carries a window descriptor qjs's bare `sp` register does not. NET HONEST CALL: with all 5 stages landed and test262 0/49775 held, ~1.3-1.5x qjs is achievable; closing the last 0.3x would require Zig-codegen-level work (custom dispatch lowering, fully raw sp everywhere) with diminishing returns and rising regression risk. The structural realignment captures ~80-85% of the 5.4x gap; the last ~15% is codegen + the alloca substitute, which are not 'structural' in the frame-representation sense.
