# QuickJS Redesign Archive

This directory stores historical redesign records that are no longer active
status sources. Archived records may explain earlier failures, fixture paths, or
handoff notes that have been superseded by current validation gates.

Active project state remains in:

- `../TRACKING.md`
- `../ERRORS_AND_LEARNINGS.md`
- `../phases/`
- `../matrices/`

Do not use archived entries as current validation evidence unless they are
reproduced and copied back into `TRACKING.md` with a fresh command, exit status,
and result summary.
